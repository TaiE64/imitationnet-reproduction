# ImitationNet 复现实验日志

> 论文:Yan et al., "Unsupervised Human-to-Robot Motion Retargeting via Shared Latent Space"(Humanoids 2023,ECHO 参考文献[22])。无官方代码,从零复现。
> 用途:ECHO 阶段 B 的地基(人-机器人共享潜空间)。
> 评测:重定向后"人 vs 机器人"的 S_RD(应远低于随机机器人)+ 逐肢体方向误差(度)。

## 方法(已实现)
- 3 个 MLP(6层×128,潜8维):Q_h(人→潜)、Q_r(机器人→潜)、D_r(潜→机器人关节角)
- 无监督:S_RD 相似度挖三元组(锚=人,正=最相似机器人,负=最不相似人)
- 3 损失:triplet(α0.05,λ10)+ rec(机器人自编码,L1,λ**5**)+ ltc(循环一致性,L1)
- 数据:机器人=TIAGo++ URDF 关节空间随机采样+FK;人=HumanML3D

### ⭐⭐⭐ 突破:可微 FK 直接损失(跳出论文的间接对齐)
**问题**:论文的对齐是间接的(triplet→latent→decode),triplet 相对 margin 本质松弛 → 顶在 82°,所有论文框架内杠杆(数据/步数/正样本)都无效。
**解法**:加直接损失 `L_fk = S_RD(R_human, FK_canon(D_r(Q_h(x_h))))`,FK 可微(pytorch_kinematics),**直接对"肢体朝向匹配"反传**,绕开 triplet 间接瓶颈。
**结果**:82°→**16.8°**(5×),方向56°→13.5°,twist26°→3°,重定向1.5×→**21×**,关节MSE 0.41→0.29(逼近论文 0.21,从2×缩到1.4×)。
**注**:16.8° **甚至低于20万库最佳匹配的19.5°**——因为 FK 损失在**连续14-DOF空间**优化,胜过任何离散采样。

### v10-v12:逼近可达性地板
- **可达性地板诊断**:逐姿态直接优化机器人关节角(从零300步)→ **4.9°**(机器人物理极限)。证明16.8°的差距是"网络没学到位",非机器人够不到。
- **v10 latent 8→32**:13.0°(放大潜空间松开瓶颈)。注:关节MSE反升(0.29→0.40),因FK直接优化朝向、自由利用7-DOF冗余→关节MSE失去意义,**肢体朝向才是真指标**。
- **v11 latent64+hidden256**:13.5°(加大网络无效→前馈平台)。
- **v12 纯FK**(去triplet/rec/ltc):**12.2°**(竞争损失轻微拖累,但仅0.8°)→ 前馈极限~12°。
- **测试时精修**(前馈初值+FK梯度下降):10步7.7° / 30步6.0° / **60步5.3°≈地板**,仍0.4ms/姿态(2.5kHz>论文1.5kHz)。

### 最终结论:82° → 5.3°(物理地板)
前馈极限~12°(摊销代价:前馈无法媲美逐姿态优化);加测试时精修达5.3°≈4.9°硬地板。**误差已触底,无法突破机器人物理约束。** 大幅偏离论文(可微FK+大潜空间+测试时精修≈学习初值的可微IK),非论文纯无监督前馈,但全程诚实记录。
**模型**:纯前馈最佳=checkpoints_imitation_purefk(12.2°,latent32);要5°级用它+测试时精修。

### ⭐ 第三个结论:~82° 是忠实方法的天花板(非数据问题)
之前用 stride 5 只取了 1/5 帧(666K)。v8 改用**完整全部帧(3.29M,5×数据)**,内存元凶 rb_rot6(1.44GB)挪到 CPU 才塞下。结果:1.51×/84°,**与 666K 的 1.56×/82° 几乎一样**。即使 rec 降到 0.16(解码器更准),重定向也没动。所有忠实杠杆(更多数据/更长训练/更紧正样本/更好解码器)都停在 ~82°。瓶颈是 **triplet 相对 margin 松弛(α0.05 不强制 z_h=z_match,对齐止于 0.22)+ 8维潜空间瓶颈**——方法固有。要再降需偏离论文(hard negative/更大潜空间/直接对齐损失/有监督配对)。

**最佳模型:v6 = checkpoints_imitation_rot/imitationnet.pt**(1.56× 超随机,全旋转82°=方向56+twist26)。对齐在 ~0.2 触底(triplet 相对 margin 的固有松弛 + 解码器精度极限);更长训练(v7)无益。论文只报告"MSE超baseline"无绝对数值,无法直接数值对标,但重定向确实 work(z_h 紧贴匹配机器人、输出合法、远超随机)。

### ⭐ 关键发现:坐标系对齐是论文未明说的核心细节
- **v2 教训**:λ_rec=5(论文值)+15M 库反而比 v1 差。论文超参不能直接迁移到简化实现;triplet 微 margin(α0.05)轻易满足≠对齐好。
- **v3-rot 失败根因**:人侧旋转(SMPL 根系,rot6d FK 组合)和机器人侧旋转(URDF base 系)**坐标系约定不同** → 同一手臂姿态 R_h≠R_r → S_RD 无意义 → 选不出正样本 → 重定向=随机(124°)。论文称全局旋转"invariant to coordinate systems",**但实际必须先对齐坐标系**。
- **v4-canon 解法**:人和机器人肢体旋转都规范化到各自**躯干坐标系**(双肩+竖直构造 body frame,`R_canon=Bᵀ·R_world`);肢体帧用**相同几何法从关节位置构造**(x=骨向,y=相邻骨臂平面→含可观测twist,z=x×y)。
  - **诊断验证**:20万机器人库,每个人姿态的最佳 canonical 匹配=**19.5°**(随机116°)→ 对齐成功,重定向可行。

### ⭐⭐ 第二个关键发现:负样本必须是"随机人",不是"最不相似的人"
- v4-canon 表示对齐了,但训练后 z_h 仍漂在"无人区"(到匹配机器人 0.41 vs 随机 0.52,几乎没对齐),D_r(z_h) 外推到 ±5 → 重定向=随机。
- **精算 triplet 数学**:`relu(d_pos − d_neg + α)`,α=0.05 极小。我之前选"512候选里最不相似的人"当负样本 → d_neg 很大 → 不等式被轻易满足(d_pos 可达 0.5)→ **根本不拉紧 z_h**。
- **论文用随机三元组**(j 是随机的另一个人)。随机负样本平均更近 → d_neg 小 → margin 有效 → z_h 被拉紧。**我之前的 argmax 负样本是不忠实的 bug。**
- 改回随机负样本(v6):对齐 0.41→**0.19**,D_r(z_h) 回到 [-1,1],重定向 1.0→**1.56×**,误差 115°→**82°**。
- **附加修复**:D_r 输出加 tanh(限到归一化关节范围 [-1,1],尊重关节限位),消除 ±5 外推导致的"比随机还差"病态。
